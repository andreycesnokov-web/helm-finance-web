## Принципы Desktop V2

1. **Правило 30 секунд.** Каждый экран должен отвечать на главный вопрос за 30 секунд или меньше.
2. **Действие важнее информации.** Показываем не просто данные — показываем что с ними делать.
3. **Финансовая ясность > красота.** При любом конфликте выбираем clarity.
4. **Desktop first.** Mobile PWA сохраняется, но приоритет проектирования — 1280px+.
5. **AI CFO — всегда рядом.** На desktop правая панель всегда видна. Она не виджет — это контекст принятия решений.

### Спецификации

**Metrics bar:**
- 4 тайла в одну строку на desktop, 2+2 строки на mobile
- Runway: цвет по риску (green ≥ 60d / amber 14–59d / red < 14d)
- Runway показывает `days_adjusted` (скорректированный с receivables/payables/payroll)

**Today's Focus:**
- Сортировка: overdue first → по due_date asc → по приоритету
- Максимум 10 items на экране
- Overdue: rose left border, icon ⚠
- Normal: чекбокс для reminders, кнопка действия для invoices/payroll
- Пустой список: "Всё под контролем ✓" — без лишних элементов

**Transaction table:**
- Колонки desktop: Date | Description | Category | Source | Scope | Amount
- Колонки mobile: свёрнутые строки (Description + Amount, Date + Source ниже)
- Сумма: tabular-nums, income зелёным, expense нейтральным (не красным)
- Показываем последние 10 транзакций + кнопка "Все →"

### Данные
- `GET /api/cfo/context` → все метрики, today_focus, AI analysis
- `GET /api/transactions?limit=10&business_id=...` → последние транзакции

### Empty State (нет транзакций)
```
        [Activity icon]
        
        Начните с добавления транзакции
        
        Добавьте первую транзакцию через
        Telegram бот или кнопку "+"
        
        [+ Добавить транзакцию]
```

### Риски
- `GET /api/cfo/context` — тяжёлый запрос. Кэшируется 5 минут (cfo_snapshots).
  При первом открытии может быть задержка ~2–3 сек. Нужен skeleton loading.
- Runway = 0 если нет транзакций — показываем "—" вместо числа
- Today's Focus mix из разных таблиц — нужен JOIN или несколько запросов с merge
